Privacy Policy

Video Speed Controller · Chrome Extension · Version 1.0.0

Video Speed Controller does not collect, store, or transmit any personal data, browsing history, or usage analytics. The extension operates entirely within your browser and never communicates with any external server.

Data Storage

All data is stored locally using the chrome.storage.local API and never leaves your device. The extension stores only what is strictly necessary for its functionality:

  • Step size preference — your chosen increment value (0.1, 0.25, or 0.5)
  • Remember-speed toggle — whether per-domain speed memory is enabled
  • Per-domain speeds — the last playback speed you set for each domain (only if the feature is enabled)
  • HUD corner positions — the corner of the video where you placed the HUD badge, per domain
  • Keyboard bindings — any custom shortcut keys you have configured

Permissions

Every permission the extension requests has a specific, limited purpose:

  • storage — saves the preferences listed above. No data is synced to your Google Account or any cloud service.
  • activeTab — allows the popup to communicate with the content script on the page you are currently viewing, only when you click the extension icon.
  • tabs — lets the background service worker query tab information so it can restore your preferred speed after a page loads.
  • scripting — declared for Manifest V3 compliance and future-proofing; currently unused at runtime.
  • <all_urls> — required to detect and control HTML5 <video> elements on any website, including embedded videos in iframes. The extension only reads video playback state; it never inspects page content, cookies, or personal information.

No Data Collection

This extension does not:

  • Collect or transmit any personal information
  • Use analytics, tracking pixels, or telemetry
  • Send data to any remote server or API
  • Access or modify cookies, passwords, or form data
  • Inject ads or modify page content beyond the speed HUD overlay
  • Share any data with third parties

Third-Party Services

This extension does not use any third-party services, libraries, or CDNs that would transmit data off-device. All code is bundled locally within the extension package.

Changes to This Policy

If this policy is updated, the version number in the extension will change and the new policy will be available at this URL. Significant changes will be noted in the extension's update description on the Chrome Web Store.
